## 1. 商模變更核心測試（每次調整必做）

每次有人提出新功能、新收費模式、新合作形式時，先回答以下三題。**任何一題答「否」，立即暫停並重新法律評估，再進入開發。**

| # | 問題 | 預期答案 |
|---|------|----------|
| 1 | 平台是否仍**未向觀眾直接收費**？（平台不向觀眾銷售代幣、訂閱、付費功能） | **是** |
| 2 | 代幣是否仍**由直播主（Host）發放**給觀眾，而非平台直接分發？ | **是** |
| 3 | 平台的收入來源是否仍**僅限直播主付的 SaaS 月費**，不含任何與遊戲廠商或觀眾互動相關的抽成或分潤？ | **是** |

**違反任一條件的後果：** 平台可能從「工具提供者」重新定性為「遊戲服務營運商」或「賭博/獎勵服務提供者」，觸發截然不同的法律框架（遊戲廠商授權、金融監管、廣告法等）。需在工程工時投入前完成法律評估。

### 1.1 常見高風險情境範例

| 提案情境 | 風險說明 | 結論 |
|---|---|---|
| 平台自己銷售「代幣包」給觀眾 | 直接違反核心設計約束 | ❌ 禁止，需法律重評 |
| 平台抽取直播主向觀眾收取的打賞分潤 | 可能使平台從工具提供者變成交易中介 | ⚠️ 需法律評估 |
| 「贊助商代幣」：廣告主付費讓觀眾獲得代幣 | 三方付費流程，可能改變工具定位 | ⚠️ 需法律評估 |
| 新增 PC 遊戲支援（Steam/Epic） | 廠商條款各異，但工具定位不變，風險較低 | ✅ 可開發，行銷用語需調整 |
| 直播主設定「最小捐款門檻才能獲得代幣」 | 屬直播主自行設定，平台不介入金流 | ✅ 符合工具定位 |

## 2. 行銷素材審查清單

每次發布公開內容（官網、社群貼文、廣告、App Store 說明）前完成以下確認。

### 2.1 商標禁用清單

以下詞彙與形象**禁止**出現在 PlayGate 品牌行銷素材中（功能性指涉除外，詳見 2.2）：

- Nintendo（任天堂）字樣、Logo、Joy-Con 外形
- Switch（作為任天堂產品專有名詞）
- PlayStation、PS5、DualSense
- Xbox、Game Pass
- 任何遊戲角色形象（Mario、Link、Pikachu 等）
- 任何遊戲截圖（除非取得正式授權）

**判斷原則：** 素材中出現這些詞彙，觀眾是否可能誤以為 PlayGate 與該廠商有官方合作？若是，禁止使用。

### 2.2 功能性指涉（允許，但需謹慎）

以下用法屬功能性說明，**允許**使用：

- "works with your game console" （適用任何主機）
- "connect your streaming PC" （PC 情境）
- "viewer-controlled live streaming" （不指明平台）
- "compatible with major gaming consoles" （不列出商標）

### 2.3 截圖使用規範

| 情境 | 規定 |
|---|---|
| 產品 UI 截圖（PlayGate 自己的介面） | ✅ 可用 |
| 遊戲畫面截圖（用於示意） | ❌ 禁止，除非取得個別授權 |
| 直播主串流畫面（遊戲內容可見） | ❌ 禁止用於官方行銷，可在直播主自己分享時轉推但不得做廣告 |
| 抽象示意圖（控制器圖示、網路拓樸圖等） | ✅ 可用，確保控制器外形不複製特定廠商專利設計 |

### 2.4 審查流程

1. 行銷人員完成素材初稿
2. 對照 [docs/marketing-guidelines.md](./marketing-guidelines.md) 逐項確認用語
3. 若含有任何第三方品名（即使是功能性描述），由產品負責人二審
4. 存檔核准紀錄（日期、審查人、版本）再發布

## 3. DMCA 流程

PlayGate 依 17 U.S.C. § 512（DMCA Safe Harbor）要求維護以下流程。

### 3.1 指定代理人資訊（Designated Agent）

|---|---|

| 郵寄地址 | [待填入 — 實體地址] |
| 版權局登記狀態 | [待確認] |

**注意：** 代理人資訊必須在官網顯眼位置公開，且與版權局登記一致。

### 3.2 收件與處理時限

```
第 0 天：收到疑似侵權通知
     ↓
第 1 個工作日：初步審查
  - 確認通知格式完整（見 3.3）
  - 若格式不完整，通知寄件人補件，重新計時
     ↓
第 2 個工作日：通知相關直播主（Host）
  - Email 說明收到通知內容（不洩露檢舉人個人資料超出法律要求）
  - 要求直播主提供說明或自行下架
     ↓
第 5 個工作日：若直播主未自行處理，平台執行下架
  - 關閉相關房間（room）
  - 若涉及持續性違規，暫停該 Host 帳號
     ↓
第 6 個工作日：通知通知人已完成處理
```

### 3.3 有效通知應包含的要素

依 17 U.S.C. § 512(c)(3)，有效侵權通知須包含：

1. 著作權人（或授權代理人）的實體或電子簽名
2. 被侵權著作的識別（標題、作者、或足以讓平台識別的描述）
3. 平台上侵權素材的位置（URL 或足夠具體的描述）
4. 通知人的聯絡資訊（地址、電話、Email）
5. 通知人善意聲明：相信該使用未經授權
6. 通知人準確性聲明：在偽證處罰下聲明以上資訊正確，且通知人有權代表著作權人行事

**格式不完整的通知：** 告知寄件人哪些要素缺失，不執行下架，等待補件。留存通訊紀錄。

### 3.4 反通知（Counter-Notice）處理

若直播主認為下架錯誤，可提交反通知：

```
直播主提交反通知
     ↓
平台審查反通知格式完整性
（須含：識別已下架素材、聲明善意誤刪、同意法院管轄、簽名）
     ↓
將反通知轉發給原通知人
     ↓
等待 10-14 個工作日
  - 若原通知人未提起訴訟 → 恢復素材
  - 若原通知人提起訴訟 → 維持下架並配合司法程序
```

### 3.5 重複侵權者政策（Repeat Infringer Policy）

依 DMCA Safe Harbor 要求，平台須維護重複侵權者終止機制：

- 同一 Host 在 12 個月內收到 **3 次** 有效 DMCA 通知：帳號暫停審查
- 暫停審查期間平台可存取通知紀錄決定是否永久終止
- 永久終止前通知 Host，給予 5 個工作日陳述機會
- 紀錄保存 3 年

## 4. 遊戲廠商風險分級表

> 此表基於公開資訊與業界觀察，非正式法律評估。具體情境應諮詢法律顧問。

| 廠商 / 平台 | 風險等級 | 主要依據 | 對 PlayGate 的實際影響 |
|---|---|---|---|
| **Nintendo** | 🔴 嚴格 | EULA 明確禁止商業使用遊戲畫面；過去積極針對 ROM 站、非官方工具發出 DMCA；但 Twitch Plays Pokémon 模式存在 12 年（2014 至今）未被取締 | 風險主要落在**直播主**（他們在使用 Nintendo 的軟體），不在平台；平台工具定位為關鍵護盾；行銷素材嚴禁使用任天堂商標 |
| **Sony（PlayStation）** | 🟡 中等 | PS 系 EULA 允許個人非商業遊玩影片分享；PSN ToS 禁止某些商業化行為；對遠端控制工具態度較 Nintendo 模糊 | 風險等級類似 Nintendo，但執法記錄較少；同樣適用工具定位護盾 |
| **Microsoft（Xbox）** | 🟡 中等 | Xbox 遊戲內容政策相對寬鬆（允許商業化內容創作）；Game Pass 有額外授權條款 | 直播主風險較低；但 Game Pass 訂閱遊戲的商業互動功能仍有疑慮，建議直播主諮詢 |
| **Steam / PC 遊戲** | 🟢 相對寬鬆 | 多數 PC 遊戲允許直播與創作者內容；各發行商政策不一，但整體執法壓力低 | PlayGate PC 支援階段風險較低；需個別確認各遊戲發行商政策 |
| **Twitch（串流平台）** | 🟡 中等 | ToS 對「自動化」和「非本人遊玩」有限制；Twitch Plays 形式長期存在並獲默許；Extension API 是更官方的互動方式 | 直播主應確認其頻道 ToS；PlayGate 模式與 Twitch Plays 類似，歷史上容忍 |
| **YouTube** | 🟢 相對寬鬆 | 對互動直播形式未特別限制；DMCA 機制活躍但針對內容而非互動工具 | 風險主要來自直播的遊戲內容，不在互動工具本身 |

**結論：** PlayGate 目前面臨的最高風險來自任天堂 EULA，但該風險主要由直播主承擔。平台的「工具提供者」定位是主要護盾。維持「平台不向觀眾收費、代幣由直播主發放」的設計約束，是保持此定位的必要條件。
